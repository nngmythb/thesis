Progressive Web Application（PWA）

漸進式網頁應用 (Progressive Web Application,PWA) 是一種融合網站與原生應用程式優勢的技術，起源可追溯至2015年，由設計師 Frances Berriman 與 Google 工程師 Alex Russell 首次提出此概念，旨在利用現代瀏覽器的新功能（如 Service Workers 和 Web App Manifest）提升網頁應用的使用體驗。然而，其技術基礎可追溯至2007年 Apple 推出 iPhone 時，曾主張以 Web 2.0 標準開發應用程式，雖然最終轉向 App Store 模式，但此理念為 PWA 奠定了初步方向。隨著 HTML5、CSS3 及 JavaScript 的進步，網頁應用逐漸具備離線使用、推播通知及主畫面安裝等功能，使其能提供接近原生應用的體驗。Google 自2016年起積極推廣 PWA，並逐步獲得 Firefox、Safari 及 Edge 等主流瀏覽器支援，成為跨平台開發的重要選項。

目標提供類似原生應用的體驗，因此會包括以下特色：

  • 漸進式
    • 透過瀏覽器使用，系統環境支援度愈高提供愈多服務，若使用老舊瀏覽器可降級顯示基本服務。使用者無須下載或安裝APP，減少消耗手機容量與記憶體，避免手機運作遲緩。
  • 逼近App體驗
    • 以行動裝置使用體驗優先設計，並自動適應任何手機、桌上型電腦與平板的螢幕尺寸。在行動裝置桌面新增圖示入口（Add to Home），下次開啟就不會出現網址列，創造逼近APP的使用介面與沉浸式體驗，讓使用者願意回訪或停留更久時間。
  • 離線使用
    • 暫時沒有網路或網路不穩仍可運作，並透過瀏覽器更新，解決App未更新造成閃退的問題。
  • 訊息推播
    • 擁有App訊息推播功能，即使沒有打開網站，也能收到訊息通知，增加企業與使用者的互動。
  • 硬體功能
    • 可調用行動裝置的硬體功能與儲存空間。
  • 增加曝光
    • 可進行搜尋引擎最佳化（Search Engine Optimization，SEO）提升瀏覽器搜尋排名，讓網頁更快被看見。
  • 提升共享性
    • 透過URL分享和使用，避免安裝App導致使用者流失或抱怨。
  • 降低成本
    • 開發者僅需維護一套程式碼，即可通用於手機、桌上型電腦與平板，省去上架費用，及重新審核的時間人力成本。

實際案例

Twitter Lite

  • 其所占大小僅原生應用程式的 1–3%

Starbucks

  • 比其 iOS 應用程式小了 99.84%
  • 部署後，線上訂單數量增加一倍
  • 桌面使用者的訂單率約與手機應用程式使用者相同

Pinterest

  • 在網站上的停留時間比之前的行動網站多了 40%
  • 廣告收益率提升 44%
  • 核心互動量增加了 60%

PWA 旅遊服務實作

目前在台灣，已上線且持續營運的 PWA 旅遊服務主要有以下：

金門行動旅服

  • 這是全國首創的 PWA 智慧旅遊平台，自 2020 年上線以來累積超過 1600 萬次瀏覽。它提供即時景點資訊、活動推薦、交通查詢、語音導覽等功能，並於 2025 年導入了 「阿特 AI 智慧客服」，支援中、英、日、韓語，讓旅客能即時互動、查詢行程與景點。
  • 每年瀏覽量近300萬次，金門日報，2024-08-30

東區觀光圈行動旅服

  • 交通部觀光署東部海岸與花東縱谷國家風景區管理處共同推出，整合行程規劃、即時天氣、交通、景點推薦、語音導覽 等功能。平台支援多語系，並導入 數位孿生導覽、720VR 實境體驗，讓旅客能深度探索花東地區。

東海岸行動旅服

  • 東區觀光圈平台的延伸提供即時天氣、航班、災害預警、景點評論等資訊，並整合 4K 景點直播、虛擬實境與數位導覽。

E-S-QUAL 服務品質量表

網站服務品質構面之相關文獻中，多數研究學者其文章的構面基礎皆來自於 SERVQUAL(服務品質) 量表，首次出現在由Parasuraman, Zeithaml, Berry 等三位作者合寫的一篇題目為《SERVQUAL A Multiple-item Scale for Measuring Consumer Perceptions of Service Quality》文章中。

爾後針對電子化服務產生，由 Parasuraman 等在2005年所建構的 E-S-QUAL 量表，共有22個測量題項，四個維度：

  1. 效率(efficiency)
  2. 履行(fulfillment)
  3. 系統可用性(system availability)
  4. 隱私(privacy)

題項

  1. 效率 (8題)
    1. 我能輕易在此網站找到我所需的資訊。
    2. 此網站可協助我輕鬆完成交易。
    3. 此網站頁面載入速度很快。
    4. 它能快速搜尋出我想要的內容。
    5. 此網站組織架構良好。
    6. 此網站導覽簡單易懂。
    7. 此網站的呈現方式清晰明瞭。
    8. 整體而言，此網站運作效率很高。
  2. 履行 (7題)
    1. 此網站能依承諾的時間完成出貨／交付。
    2. 我從此網站訂購的商品準時到達。
    3. 此網站所承諾有庫存的商品確實有存貨。
    4. 此網站的訂單追蹤功能運作正常。
    5. 此網站開立的帳單皆屬正確。
    6. 此網站會立即發送訂單確認通知。
    7. 此網站能直接且完整地履行我的訂單。
  3. 系統可用性 (4題)
    1. 此網站隨時都能提供服務，不會無故中斷。
    2. 此網站幾乎從不當機。
    3. 此網站能立刻啟動並且順暢運行。
    4. 此網站整體運作可靠、穩定。
  4. 隱私 (3題)
    1. 此網站充分保護我所提供的資訊。
    2. 此網站確保我的個人資訊受到妥善保護。
    3. 此網站對於我的個人資訊隱私給予明確承諾並予以保障。
